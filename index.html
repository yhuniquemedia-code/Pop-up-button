Email Capture → Redirect → Follow-Up Flow

This document explains the full email capture and redirect system for your website. Your coder will know exactly what it does, why, and how to implement it. You will provide the actual sales page link later.

1. Purpose of the Code

  • Capture visitor emails: immediately after they show interest using a micro-commitment step.
  • Redirect after submission: send visitors to the next step (sales page, upsell, or main offer).
  • Feed emails into your platform: stored for future automation (Mailchimp, ConvertKit, Selar, etc.).
  • Maintain user focus: ensures visitor attention is not diverted before they commit.

2. Core Components of the Code

A. Micro-commitment button

Purpose: Gives the visitor a small action they can take immediately, increasing engagement.

Behavior: Visitor clicks a button like Get My 21-Day Study System which triggers the email popup.

<button id="getNowBtn">Get My 21-Day Study System</button>

B. Email capture popup

Purpose: Collect the visitor’s email before redirecting them to the next step.

Behavior: Shows a modal popup with a simple form (email input + submit). Only essential info is requested.

<div id="emailPopup" style="display:none;">
  <form id="emailForm">
    <input type="email" name="email" placeholder="Enter your email" required>
    <button type="submit">Submit</button>
  </form>
</div>

Functionality:

  1. Email is sent to your platform when "submit" is clicked.
  2. Visitor is redirected automatically after submission.

C. Submit handler & redirect

Purpose: Connect the email capture to the next step.

Behavior: Submits email using AJAX/API and redirects visitor to next page.

document.getElementById('emailForm').addEventListener('submit', function(e){
  e.preventDefault(); // prevent default form submission
  let email = this.email.value;
  
  // Send email to your platform (example with API call)
  // fetch('https://your-email-platform.com/add', { method:'POST', body: { email } });
  
  // Redirect after successful submission
  window.location.href = "https://yhuniquemedia-code.github.io/study-habit-systems/"; // <-- Replace this
});

D. Why it’s done this way

  • Micro-commitment first: reduces friction and increases email submissions.
  • Popup form for focus: keeps email capture separate, prevents distraction.
  • Automatic redirect: smooth flow from interest → email → next step.
  • Future automation ready: enables nurture emails, upsells, cart abandonment follow-ups.

E. Placement on the page

  • Micro-commitment button: above-the-fold.
  • Email popup: hidden initially, triggered by button click.
  • Redirect URL: dynamic, inserted in JS code after submission.

F. What the coder needs to do

  1. Place the button in the desired spot in HTML.
  2. Add the popup form somewhere in HTML (hidden with CSS).
  3. Add JS code to trigger popup, submit email, and redirect after submission.
  4. Make the redirect URL dynamic for easy updates.

✅ Summary

  • Efficient lead capture.
  • Keeps visitors focused on the offer.
  • Automatic redirect after submission.
  • Sets up future email marketing funnel and upsells.
